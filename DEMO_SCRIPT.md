# Demo Script (≈2 minutes)

## 0:00 – Introduction

> "Hi, I'm LayLa, and this is ForgeOS.

> ForgeOS is an AI-powered software development workspace that uses multiple specialized AI agents to transform an idea into a structured software project. Instead of relying on one large AI prompt, ForgeOS assigns different responsibilities to dedicated agents that collaborate throughout the development process."

---

## 0:20 – Generate a Project

Enter:

> **Build a modern coffee shop ordering and loyalty app with mobile ordering, rewards, personalized recommendations, and inventory management.**

Click **Launch**.

> "With a single prompt, ForgeOS creates a complete project workspace. While the agents collaborate, you can watch their progress in real time."

---

## 0:40 – Agent Society

Point to the AI Agent panels.

> "Each agent has a specialized role. The CEO defines the vision, the Product Manager creates the roadmap, the Engineer develops the solution, QA validates it, Security reviews it, the Reviewer checks overall quality, and DevOps prepares the final deliverable.

> Rather than one AI doing everything, ForgeOS demonstrates collaborative AI agents working together."

> "As the project progresses, ForgeOS continuously updates the workspace with generated plans, documentation, project artifacts, and implementation details.

# Common Judge Questions

### Why multiple agents?

Each agent has a dedicated responsibility, allowing planning, implementation, review, security, and deployment to be handled separately instead of by one large prompt.

---

### What happens if the AI service is unavailable?

ForgeOS automatically switches into deterministic demo mode, allowing the full workflow to continue without requiring an API key.

---

### What technologies did you use?

- Next.js
- React
- TypeScript
- FastAPI
- Python
- WebSockets
- Qwen API
- Vercel
- Render
- UptimeRobot

---

### Is this a real deployment?

Yes. The frontend is deployed on Vercel, the backend is deployed on Render, and UptimeRobot helps keep the backend responsive by reducing cold-start delays.
